Who are they?
›

Cloudflare คือ connectivity cloud company — ให้บริการ network security, performance, และ developer platform ผ่าน global network ที่มี 300+ data centers ใน 100+ ประเทศ เป็นหนึ่งใน most densely distributed networks บนโลก

ถ้าจะเข้าใจ Cloudflare ต้องเข้าใจ concept ว่า บริษัทคือ "security + connectivity + compute layer" ที่ sit ตรงกลางระหว่าง internet กับ customers ของ Cloudflare — ทุก traffic ผ่าน Cloudflare จะ inspected, filtered, optimized ก่อนถึง origin server ทำให้ add value ได้เยอะมากพร้อมกัน

02
Journey
›

Cloudflare ก่อตั้งด้วย vision ว่า internet security ต้องเป็น commodity ที่ทุกคนเข้าถึงได้ — ไม่ใช่ luxury ของ enterprise ใหญ่เท่านั้น journey จาก DDoS mitigation เล็กๆ สู่ connectivity cloud platform ขนาดใหญ่

2009
ก่อตั้งโดย Matthew Prince, Michelle Zatlyn, Lee Holloway: เริ่มจาก Project Honey Pot (spam tracking) และ idea ว่าสามารถ offer security-as-a-service ผ่าน internet โดยไม่ต้องติดตั้ง hardware บน-premise เข้า Y Combinator รุ่น Summer 2009
2010
Launch สู่ Public: เปิดตัว beta ต่อ public ด้วย free tier — anyone สามารถ protect website ด้วย Cloudflare โดยเพียงแค่ change DNS ไม่ต้องเปลี่ยน hardware ใดๆ model นี้สร้าง massive user base อย่างรวดเร็ว
2012–2016
Scale Free Tier + Build Enterprise: Cloudflare expand network ไปทั่วโลกอย่างรวดเร็ว โดย fund expansion ด้วย venture capital (Sequoia, Google Ventures ฯลฯ) เริ่ม convert free users เป็น paid enterprise customers ด้วย advanced security features
2019
IPO บน NYSE — $3.5B Valuation: List บน NYSE ในราคา $15/share ที่ market cap ~$5B ณ วัน IPO Cloudflare reveal ครั้งแรกว่ามี revenue ใกล้ $300M และ growing เร็วมาก ขณะที่ยังขาดทุน
2020–2021
Covid Acceleration + SASE Platform Pivot: Remote work drove cybersecurity demand สูงมาก Cloudflare launch Zero Trust / SASE products (Cloudflare Access, Magic WAN) เปลี่ยนจาก CDN/DDoS company เป็น network security platform stock ขึ้น 6x ใน 2 ปี
2023–2025
AI Inference Edge + Workers AI: Cloudflare launch Workers AI (run AI models ที่ edge ใกล้ users) และ AI Gateway เป็น infrastructure สำหรับ AI applications ที่ต้องการ low-latency inference บริษัทเริ่ม re-position เป็น AI infrastructure company ด้วย

05
What do they Actually Do?
›

Cloudflare ทำงานเป็น "intermediary layer" ระหว่าง internet กับ customers — เมื่อ user ทั่วโลก request ไปยัง website ที่ใช้ Cloudflare request จะถูก route ผ่าน Cloudflare network ก่อน ซึ่ง inspect, filter, cache, accelerate แล้วค่อยส่งต่อ

Network Services (CDN + Performance + DDoS): ลดเวลา load website ด้วย edge caching, DDoS protection ป้องกัน volumetric attacks สูงถึง Tbps, DNS ที่เร็วที่สุด (1.1.1.1 — fastest public resolver), Load balancing, Bot management

Zero Trust / SASE Security: Cloudflare Access (ควบคุมว่าใคร access อะไรได้), Magic WAN (replace erbate SD-WAN), Cloudflare Gateway (secure DNS + HTTP filtering), CASB, DLP — ทั้งหมดนี้รวมกันเป็น Zero Trust Network Access (ZTNA) platform แทน traditional VPN

Developer Platform (Workers, R2, AI): Workers (serverless compute at edge — run JavaScript/Python ใกล้ users ทั่วโลก), R2 (object storage คล้าย S3 แต่ไม่มี egress fee), Workers AI (run AI models at edge), AI Gateway (proxy + caching สำหรับ LLM calls) — segment นี้เป็น future growth engine

How they Compound
›

Cloudflare compound ผ่าน network effects ที่แข็งแกร่งมาก — ยิ่ง network ใหญ่ขึ้น ยิ่ง protect ดีขึ้น (เห็น attack patterns เยอะขึ้น) ยิ่งเร็วขึ้น (peering ดีขึ้น) ทำให้ network ยิ่ง valuable ยิ่งขึ้นในวงกลม

Threat Intelligence Network Effect: ทุก traffic ที่ผ่าน Cloudflare (หลาย Trillion requests/day) ทำให้ Cloudflare เห็น attack patterns ก่อน competitors เปลี่ยน DDoS signatures, bot patterns, phishing domains ก่อนที่จะ spread ไปยัง internet ทั่วไป — ยิ่ง network ใหญ่ยิ่ง intelligence ดี

Platform Expansion Optionality: ทุก product ใหม่ที่ Cloudflare launch (R2 storage, Workers AI, AI Gateway, Queues) ใช้ network เดิมที่มีอยู่แล้ว ดังนั้น incremental cost ต่ำมากแต่ TAM expand ทุกครั้ง เหมือน Cloudflare "unlock" new revenue streams โดยใช้ existing infrastructure

Free Tier as Moat: Cloudflare มี ~35M+ websites ใช้ free tier อยู่ ซึ่ง (1) สร้าง massive developer affinity (2) feed data ให้ improve services (3) create pipeline สำหรับ enterprise conversion บริษัทอื่นทำแบบนี้ไม่ได้ เพราะต้นทุน network infrastructure สูงมาก

08
Current Situation & Positioning
›

Cloudflare อยู่ใน transition จาก "network security company" สู่ "AI infrastructure layer" — timing ของ AI boom ดีมากเพราะ Cloudflare network เหมาะอย่างมากสำหรับ low-latency AI inference at edge

📈 Momentum

Large Customer Growth Acceleration

$100k+ customers เพิ่ม +26% YoY เป็น 3,265 รายใน FY2024 — enterprise adoption เร็วขึ้น ซึ่ง drives higher ASP และ better NRR เหล่านี้เป็น multi-product customers ที่ sticky มาก

📈 Momentum

Workers AI / AI Gateway — AI Inference Edge

Developer นิยม deploy AI inference บน Cloudflare Workers เพราะ latency ต่ำกว่า cloud (ใกล้ users มากกว่า) และ no egress fees Workers AI เป็น fastest growing product ในประวัติบริษัท — แม้ contribution ต่อ revenue ยังเล็กอยู่ใน FY2024

👁 จับตา

Competition จาก AWS + Azure Zero Trust

AWS และ Microsoft กำลัง push Zero Trust / SASE products ของตัวเอง ลูกค้าที่อยู่ใน AWS/Azure ecosystem อาจ prefer native solutions แม้ Cloudflare ยังดีกว่าใน features ปัจจุบัน

👁 จับตา

Valuation Premium ต้องการ Execution Perfect

Cloudflare trade ที่ premium valuation (P/S ~20x+) ซึ่ง embed high-growth expectation ใดๆ ที่ growth slowdown จาก 28% → 20% จะทำให้ multiple compress หนัก ต้อง execution perfect ทุกไตรมาส

⚠️ Risk

GAAP EPS ยังเป็นลบ — SBC Dilution

GAAP loss ส่วนใหญ่มาจาก stock-based compensation (SBC) ที่สูง จาก equity-heavy compensation culture ซึ่งหมายความว่า shareholder ถูก dilute ทุกปีในขณะที่ management ได้รับ compensation ใหญ่
